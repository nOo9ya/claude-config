# Title type : can be - Conventional Commits styles
# feat       : 새로운 기능 추가(feature)
# fix        : 버그 수정(bug fix)
# docs       : 문서 수정(README, API 문서 수정)
# style      : 코드 의미에 영향을 주지 않는 css등 사용자 ui 디자인 변경
# refactor   : 코드 리팩토링(기능 변경 없음)
# comment    : 주석 추가 및 수정
# security   : 보안 취약점 해결 및 관련 변경 사항
# perf       : 성능 개선 관련 수정
# test       : 테스트 코드 추가 및 테스트 코드 리팩토링
# chore      : 빌드 프로세스 수정 및 환경 설정 파일 변경, 패키지 매니저 수정사항
# rename     : 파일 혹은 폴더명을 수정하거나 옮기는 작업만 하는 경우
# remove     : 파일을 삭제하는 작업만 수행한 경우
# revert     : 이전 커밋으로 되돌리는 작업(예시: This reverts commit <hash>)
# ci         : CI 구성 파일 및 스크립트에 대한 변경 사항(예시 범위: Travis, Circle, BrowserStack, SauceLabs)
# build      : 빌드 시스템이나 외부 종속성에 영향을 미치는 변경 사항(예시 범위: gulp, broccoli, npm)
